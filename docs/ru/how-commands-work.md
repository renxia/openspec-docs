# Как работают команды

**Единственное, что нужно знать: OpenSpec имеет два типа команд, и они выполняются в двух разных местах.**

- Команды `openspec ...` запускаются в вашем **терминале**. (Пример: `openspec init`.)
- Команды `/opsx:...` запускаются в **чате вашего AI-ассистента**. (Пример: `/opsx:propose`.)

Если вы когда-либо вводили `/opsx:propose` в терминале, а ничего не происходило, эта страница объясняет почему. Вы обращаетесь к неправильной части OpenSpec. Слэш-команды — это не команды терминала. Это инструкции, которые вы даете своему AI-ассистенту для кодирования в том же чате, где обычно пишете "добавить форму входа".

Это одно различие является самой распространенной камнем преткновения для новых пользователей, поэтому давайте сделаем его кристально ясным.

## Две части

OpenSpec — это один проект, выполняющий две роли.

**CLI (терминальная часть).** Это программа под названием `openspec`, которую вы устанавливаете и запускаете из своей оболочки. Она настраивает ваш проект, перечисляет и проверяет изменения, показывает дашборд и архивирует завершенную работу. Вы вводите их в iTerm, терминал VS Code, PowerShell — везде, где бы вы ни запускали `git` или `npm`.

```bash
openspec init        # настройка OpenSpec для этого проекта
openspec list        # просмотр активных изменений
openspec view        # открытие интерактивного дашборда
```

**Слэш-команды (чат-часть).** Короткие команды, такие как `/opsx:propose` и `/opsx:apply`, которые вы вводите в своего AI-ассистента. Они сообщают ИИ следовать рабочему процессу OpenSpec: составить предложение, написать спецификации, построить на основе списка задач, заархивировать по завершении. Вы вводите их в Claude Code, Cursor, Windsurf, Copilot или любого другого ассистента.

```text
/opsx:propose add-dark-mode    (введено в чат AI)
/opsx:apply                    (введено в чат AI)
/opsx:archive                  (введено в чат AI)
```

Вот ментальная модель на одной картинке:

```text
        ВАШ ТЕРМИНАЛ                         ЧАТ ВАШЕГО AI-АССИСТЕНТА
   ┌──────────────────────┐               ┌──────────────────────────────┐
   │  $ openspec init     │   устанавливает    │  /opsx:propose add-dark-mode  │
   │  $ openspec list     │  ──────────►  │  /opsx:apply                  │
   │  $ openspec view     │   команды        │  /opsx:archive                │
   └──────────────────────┘    & навыки   └──────────────────────────────┘
        запускайте openspec здесь                       запускайте /opsx:* здесь
```

Обратите внимание на стрелку. Запуск `openspec init` в вашем терминале *устанавливает* слэш-команды в ваш AI-инструмент. Терминальная часть настраивает чат-часть. После этого повседневная работа ведется в основном в чате.

## «Как мне запустить интерактивный режим?»

**Отдельного интерактивного режима запуска нет.** Этот вопрос часто возникает, поэтому он заслуживает простого ответа.

Вы не входите в специальный режим OpenSpec. Вы просто открываете своего AI-кодирующего ассистента, как обычно, и вводите слэш-команду в чат. Слэш-команда *является* тем способом, которым вы «входите» в OpenSpec. Ваш ассистент распознает ее, загружает соответствующий навык (skill) OpenSpec и начинает следовать рабочему процессу.

Итак, реальные инструкции следующие:

1. Откройте своего AI-кодирующего ассистента (Claude Code, Cursor, Windsurf и т. д.) в вашем проекте.
2. Введите `/opsx:propose` в чат, там же, где вы вводите любой другой запрос.
3. Наблюдайте за автодополнением: если OpenSpec установлен, при наборе слэша вы увидите `/opsx:propose`, `/opsx:apply` и других.

Вот и всё. Не нужно переключать режим, запускать демона или иметь отдельное окно.

Одна вещь, которая действительно интерактивна, существует в терминале: `openspec view`. Он открывает дашборд для просмотра ваших спецификаций и изменений. Но это просмотрщик, а не то, с помощью чего вы предлагаете и строите. Построение происходит через слэш-команды в чате.

## Почему существует это разделение

Стоит понять, потому что это объясняет, почему OpenSpec работает с более чем 25 разными AI-инструментами.

CLI — это **двигатель**. Он знает правила: как выглядит папка изменений, какие артефакты зависят от каких, как слить дельта-спецификацию в вашу истину (source of truth). Это одинаково везде.

Слэш-команды — это **рулевое колесо**, и у каждого AI-инструмента оно немного разное. Claude Code называет их командами. Cursor и Windsurf имеют свои форматы. Некоторые инструменты называют их навыками (skills). Когда вы запускаете `openspec init`, OpenSpec генерирует нужный файл для каждого выбранного вами инструмента, чтобы намерение `/opsx:propose` работало независимо от того, какого ассистента вы предпочитаете.

Сила этого дизайна заключается в том, что вы изучаете рабочий процесс один раз и применяете его ко всем инструментам. Компромисс: точный синтаксис команды может немного различаться между инструментами, что будет рассмотрено в следующем разделе.

## Синтаксис слэш-команд по инструментам

Намерение (intent) одинаково везде. Пунктуация отличается. Используйте форму, соответствующую вашему ассистенту.

| Tool | Как вы это вводите |
|------|-----------------|
| Claude Code | `/opsx:propose`, `/opsx:apply` |
| Cursor | `/opsx-propose`, `/opsx-apply` |
| Windsurf | `/opsx-propose`, `/opsx-apply` |
| GitHub Copilot (IDE) | `/opsx-propose`, `/opsx-apply` |
| Kimi CLI | skill-style, например `/skill:openspec-propose` |
| Trae | skill-style, например `/openspec-propose` |

Большинство инструментов используют либо формат с двоеточием (`/opsx:propose`), либо формат с дефисом (`/opsx-propose`). Несколько инструментов отображают OpenSpec как именованные навыки (skills), а не как слэш-команды; для них вы вызываете навык по имени. Полный список по инструментам, включая точные пути файлов, где что должно быть записано, находится в [Supported Tools](supported-tools.md).

Если сомневаетесь, введите слэш в чат своего AI и посмотрите на автодополнение. Ваш инструмент покажет вам ожидаемый формат.

## Как команды появились: навыки и команды

Когда вы запускаете `openspec init` (или `openspec update`), OpenSpec записывает небольшие файлы в ваш проект, чтобы ваш AI-инструмент мог найти рабочий процесс. В зависимости от вашего инструмента и настроек, это **навыки** (skills), **команды** или оба.

- **Навыки (Skills)** находятся в таких местах, как `.claude/skills/openspec-*/SKILL.md`. Это зарождающийся кросс-инструментальный стандарт: папка инструкций, которую автоматически обнаруживает ваш ассистент.
- **Команды** находятся в таких местах, как `.claude/commands/opsx/<id>.md`. Это более старые файлы слэш-команд для конкретного инструмента.

Вам не нужно беспокоиться о том, какой из них использует ваш инструмент. Вы просто вводите слэш-команду, и она работает. Но знание того, что эти файлы существуют, помогает, когда что-то идет не так: если ваши команды исчезли, это обычно означает, что эти файлы отсутствуют или устарели, а `openspec update` их регенерирует.

См. [Supported Tools](supported-tools.md) для точных путей по инструментам и [Migration Guide](migration-guide.md), как навыки заменили старый подход, основанный только на командах.

## Подтверждение установки

Быстрая проверка, сначала самые быстрые:

1. **Введите слэш в чате своего AI.** Начните вводить `/opsx` и следите за предложениями автодополнения. Если они появляются, вы готовы.
2. **Найдите файлы.** Для Claude Code проверьте, содержит ли `.claude/skills/` папки `openspec-*`. Другие инструменты используют свои собственные каталоги ([Supported Tools](supported-tools.md) перечисляет их).
3. **Повторно запустите настройку.** Из корневой директории проекта выполните `openspec update`. Это регенерирует файлы навыков и команд для всех настроенных вами инструментов.
4. **Перезапустите ассистента.** Многие инструменты сканируют наличие навыков и команд при запуске, поэтому свежее окно может быть недостающим шагом.

## Какие команды у меня есть?

По умолчанию OpenSpec устанавливает базовый набор слэш-команд:

- `/opsx:explore`: обдумать идею с ИИ перед тем, как вносить изменения (отличный первый шаг, когда вы не уверены)
- `/opsx:propose`: создать изменение и составить все его планирующие артефакты за один раз
- `/opsx:apply`: построить изменение, пройдя через список задач
- `/opsx:sync`: слить обновления спецификаций изменения в ваши основные спецификации (обычно автоматически)
- `/opsx:archive`: завершить изменение и заархивировать его

Хороший стандартный ритм: `explore`, когда вы выясняете, что делать, затем `propose`, `apply`, `archive`. Руководство [Explore First](explore.md) объясняет, почему этот начальный шаг окупается.

Существует также **расширенный** набор для тех, кто хочет более тонкий контроль (`/opsx:new`, `/opsx:continue`, `/opsx:ff`, `/opsx:verify`, `/opsx:bulk-archive`, `/opsx:onboard`). Вы включаете его с помощью `openspec config profile`, а затем применяете с помощью `openspec update`.

Новичок во всем этом? `/opsx:onboard` (из расширенного набора) проведет вас через полное изменение в вашем собственном кодовом проекте, комментируя каждый шаг. Это максимально дружелюбное знакомство.

Для детального описания того, что делает каждая команда, см. [Commands](commands.md). Для того, когда какую использовать, см. [Workflows](workflows.md).

## Чистый первый запуск

Подводя итог, вот вся последовательность с обозначением каждого шага по месту его выполнения.

```text
TERMINAL   $ npm install -g @fission-ai/openspec@latest
TERMINAL   $ cd your-project
TERMINAL   $ openspec init
              (устанавливает слэш-команды в ваш AI-инструмент)

AI CHAT      /opsx:explore
              (опционально: обдумать идею с ИИ перед тем, как приступить)

AI CHAT      /opsx:propose add-dark-mode
              (AI составляет предложение, спецификации, дизайн, задачи)

AI CHAT      /opsx:apply
              (AI строит его, отмечая выполненные задачи)

AI CHAT      /opsx:archive
              (изменение сливается в ваши спецификации и архивируется)
```

Два терминальных шага для настройки. А затем вы работаете в чате. Таков ритм.
