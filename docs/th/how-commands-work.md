# วิธีการทำงานของคำสั่ง

**สิ่งที่สำคัญที่สุดที่ต้องทราบคือ: OpenSpec มีสองประเภทของคำสั่ง และทำงานในสองสถานที่ที่แตกต่างกัน**

- คำสั่งประเภท `openspec ...` ทำงานใน **เทอร์มินัล** ของคุณ (ตัวอย่าง: `openspec init`)
- คำสั่งประเภท `/opsx:...` ทำงานใน **แชทของผู้ช่วย AI** ของคุณ (ตัวอย่าง: `/opsx:propose`)

หากคุณเคยพิมพ์ `/opsx:propose` ในเทอร์มินัลและไม่เกิดอะไรขึ้น หน้านี้คือเหตุผล คุณกำลังติดต่อกับครึ่ง的一半ที่ผิดของ OpenSpec คำสั่งแบบ Slash ไม่ใช่คำสั่งเทอร์มินัล เป็นคำสั่งที่คุณส่งให้ผู้ช่วยเขียนโค้ด AI ในกล่องแชทเดียวกับที่คุณมักจะพิมพ์ "เพิ่มแบบฟอร์มเข้าสู่ระบบ" อยู่

ความแตกต่างเพียงอย่างนี้เป็นอุปสรรคที่พบบ่อยที่สุดสำหรับผู้ใช้ใหม่ ดังนั้นเราจะอธิบายให้ชัดเจนที่สุด

## สองครึ่ง的一半ของ OpenSpec

OpenSpec เป็นโปรเจกต์เดียวที่สวมสองหมวก

**CLI (ครึ่ง的一半ของเทอร์มินัล)** เป็นโปรแกรมชื่อ `openspec` ที่คุณติดตั้งและรันจากเชลล์ของคุณ โปรแกรมนี้จะตั้งค่าโปรเจกต์ของคุณ แสดงรายการและตรวจสอบความถูกต้องของการเปลี่ยนแปลง แสดงแดชบอร์ด และเก็บไฟล์งานที่เสร็จสมบูรณ์แล้ว คุณพิมพ์คำสั่งเหล่านี้ใน iTerm เทอร์มินัลของ VS Code PowerShell หรือทุกสถานที่ที่คุณเคยรัน `git` หรือ `npm` อยู่

```bash
openspec init        # ตั้งค่า OpenSpec ในโปรเจกต์นี้
openspec list        # ดูการเปลี่ยนแปลงที่กำลังดำเนินการอยู่
openspec view        # เปิดแดชบอร์ดแบบโต้ตอบ
```

**คำสั่ง Slash (ครึ่ง的一半ของแชท)** เป็นคำสั่งสั้นเช่น `/opsx:propose` และ `/opsx:apply` ที่คุณพิมพ์ในผู้ช่วย AI ของคุณ คำสั่งเหล่านี้จะบอกให้ AI ทำตามเวิร์กโฟลว์ของ OpenSpec: เขียนร่างข้อเสนอ เขียนสเปค สร้างจากรายการงาน และเก็บไฟล์เมื่อเสร็จสิ้น คุณพิมพ์คำสั่งเหล่านี้ใน Claude Code Cursor Windsurf Copilot หรือผู้ช่วย AI ทุกตัวที่คุณใช้งาน

```text
/opsx:propose add-dark-mode    (พิมพ์ในแชทผู้ช่วย AI ของคุณ)
/opsx:apply                    (พิมพ์ในแชทผู้ช่วย AI ของคุณ)
/opsx:archive                  (พิมพ์ในแชทผู้ช่วย AI ของคุณ)
```

นี่คือโมเดลการทำงานในภาพรวมหนึ่งภาพ:

```text
        เทอร์มินัลของคุณ                         แชทของผู้ช่วย AI ของคุณ
   ┌──────────────────────┐               ┌──────────────────────────────┐
   │  $ openspec init     │    ติดตั้ง      │  /opsx:propose add-dark-mode  │
   │  $ openspec list     │  ──────────►  │  /opsx:apply                  │
   │  $ openspec view     │  คำสั่งและสกิล │  /opsx:archive                │
   └──────────────────────┘               └──────────────────────────────┘
        รัน openspec ที่นี่                       รัน /opsx:* ที่นี่
```

โปรดสังเกตลูกศร การรัน `openspec init` ในเทอร์มินัลของคุณคือสิ่งที่ *ติดตั้ง* คำสั่ง Slash ลงในเครื่องมือ AI ของคุณ ครึ่ง的一半ของเทอร์มินัลจะตั้งค่าครึ่ง的一半ของแชทให้พร้อมใช้งาน หลังจากนั้น การใช้งานประจำวันส่วนใหญ่จะเกิดขึ้นในแชท

## "ฉันจะเริ่มโหมดโต้ตอบได้อย่างไร?"

**ไม่มีโหมดโต้ตอบแยกต่างหากที่ต้องเริ่มต้น** คำถามนี้พบบ่อยมาก ดังนั้นเราจะตอบอย่างตรงไปตรงมา

คุณไม่ต้องเข้าสู่โหมดพิเศษของ OpenSpec เพียงแค่เปิดผู้ช่วยเขียนโค้ด AI ของคุณตามปกติ แล้วพิมพ์คำสั่ง Slash ลงในแชท คำสั่ง Slash นี้ *คือวิธี* ที่คุณ "เข้าสู่" OpenSpec ผู้ช่วยของคุณจะรู้จักคำสั่งนี้ โหลดสกิลของ OpenSpec ที่ตรงกัน และเริ่มทำตามเวิร์กโฟลว์ทันที

ดังนั้น คำแนะนำที่แท้จริงคือ:
1. เปิดผู้ช่วยเขียนโค้ด AI ของคุณ (Claude Code, Cursor, Windsurf และอื่นๆ) ในโปรเจกต์ของคุณ
2. พิมพ์ `/opsx:propose` ในแชทของมัน ที่เป็นสถานที่เดียวกับที่คุณพิมพ์คำขออื่นๆ
3. ดูฟีเจอร์เติมคำอัตโนมัติ: หาก OpenSpec ถูกติดตั้งแล้ว คุณจะเห็น `/opsx:propose` `/opsx:apply` และคำสั่งอื่นๆ ปรากฏขึ้นขณะที่คุณพิมพ์เครื่องหมาย Slash

นั่นก็คือทั้งหมด ไม่ต้องสลับโหมด ไม่ต้องเปิด daemon ไม่ต้องเปิดหน้าต่างแยก

มีสิ่งหนึ่งที่ *เป็นโต้ตอบจริงๆ* อยู่ในเทอร์มินัลคือ `openspec view` คำสั่งนี้จะเปิดแดชบอร์ดสำหรับดูสเปคและการเปลี่ยนแปลงของคุณ แต่เป็นเพียงเครื่องมือดู ไม่ใช่สิ่งที่คุณใช้เพื่อเสนอและสร้างงาน การสร้างงานทั้งหมดเกิดขึ้นผ่านคำสั่ง Slash ในแชท

## ทำไมจึงมีการแบ่งส่วนเช่นนี้

มันคุ้มค่าที่จะเข้าใจ เพราะมันอธิบายว่าเพราะเหตุใด OpenSpec ทำงานร่วมกับเครื่องมือ AI มากกว่า 25 ตัว

CLI คือ **เครื่องยนต์** มันทราบกฎทั้งหมด: รูปลักษณ์ของโฟลเดอร์การเปลี่ยนแปลง อาร์ติแฟกต์ต่างๆ ขึ้นกับกันอย่างไร และวิธีการผสานสเปค delta เข้ากับแหล่งข้อมูลหลักของคุณ มันเหมือนกันทุกแห่ง

คำสั่ง Slash คือ **พวงมาลัย** และเครื่องมือ AI ทุกตัวมีพวงมาลัยที่แตกต่างกันเล็กน้อย Claude Code เรียกมันว่า commands Cursor และ Windsurf มีรูปแบบของตัวเองบ้าง เครื่องมือบางตัวเรียกมันว่า skills เมื่อคุณรัน `openspec init` OpenSpec จะสร้างไฟล์ประเภทที่เหมาะสมสำหรับแต่ละเครื่องมือที่คุณเลือก ดังนั้นความตั้งใจเดียวกันของ `/opsx:propose` ทำงานได้ไม่ว่าคุณใช้ผู้ช่วยตัวไหน

จุดแข็งของการออกแบบนี้: คุณเรียนรู้เวิร์กโฟลว์เพียงครั้งเดียว และนำไปใช้กับเครื่องมือต่างๆ ได้ทั้งหมด จุดอ่อน: ไวยากรณ์ที่แม่นยำของคำสั่งอาจแตกต่างกันเล็กน้อยระหว่างเครื่องมือ ซึ่งจะเป็นเนื้อหาในส่วนต่อไป

## ไวยากรณ์คำสั่ง Slash ตามเครื่องมือ

ความตั้งใจเหมือนกันทุกแห่ง แต่เครื่องหมายวรรคต่างกัน ใช้รูปแบบที่ตรงกับผู้ช่วยของคุณ

| เครื่องมือ | วิธีพิมพ์ |
|------|-----------------|
| Claude Code | `/opsx:propose`, `/opsx:apply` |
| Cursor | `/opsx-propose`, `/opsx-apply` |
| Windsurf | `/opsx-propose`, `/opsx-apply` |
| GitHub Copilot (IDE) | `/opsx-propose`, `/opsx-apply` |
| CodeArts | รูปแบบสกิล ตัวอย่าง `/openspec-propose` |
| Codex | รูปแบบสกิล ผ่าน `.codex/skills/openspec-*` |
| Oh My Pi | `/opsx-propose`, `/opsx:apply` |
| Kimi CLI | รูปแบบสกิล ตัวอย่าง `/skill:openspec-propose` |
| Trae | `/opsx-propose`, `/opsx-apply` |

เครื่องมือส่วนใหญ่ใช้รูปแบบเครื่องหมาย colon (`/opsx:propose`) หรือรูปแบบเครื่องหมายขีด (`/opsx-propose`) เครื่องมือบางตัวแสดง OpenSpec เป็นสกิลที่มีชื่อแทนคำสั่ง Slash สำหรับเครื่องมือเหล่านี้คุณเรียกใช้สกิลด้วยชื่อ รายชื่อเครื่องมือทั้งหมดรวมถึงตำแหน่งที่ไฟล์ต่างๆ ถูกเขียนอยู่จริงๆ สามารถดูได้ที่ [เครื่องมือที่รองรับ](supported-tools.md)

หากไม่แน่ใจ ให้พิมพ์เครื่องหมาย Slash ในแชทผู้ช่วย AI ของคุณและดูที่ฟีเจอร์เติมคำอัตโนมัติ เครื่องมือของคุณจะแสดงรูปแบบที่คาดหวังไว้ให้คุณดู

## คำสั่งมาจากจากไหน: สกิลและคำสั่ง

เมื่อคุณรัน `openspec init` (หรือ `openspec update`) OpenSpec จะเขียนไฟล์ขนาดเล็กลงในโปรเจกต์ของคุณเพื่อให้เครื่องมือ AI ของคุณค้นหาเวิร์กโฟลว์ได้ ขึ้นอยู่กับเครื่องมือและการตั้งค่าของคุณ ไฟล์เหล่านี้จะเป็น **สกิล** **คำสั่ง** หรือทั้งสองอย่าง

- **สกิล** อยู่ในตำแหน่งเช่น `.claude/skills/openspec-*/SKILL.md` เป็นมาตรฐานข้ามเครื่องมือที่กำลังเกิดขึ้น: โฟลเดอร์ที่มีคำสั่งที่ผู้ช่วยของคุณตรวจพบโดยอัตโนมัติ
- **คำสั่ง** อยู่ในตำแหน่งเช่น `.claude/commands/opsx/<id>.md` เป็นไฟล์คำสั่ง Slash เก่ากว่าเฉพาะเครื่องมือ Codex จะไม่ได้รับไฟล์คำสั่งที่สร้างขึ้นโดยอัตโนมัติ ใช้ `.codex/skills/openspec-*` แทน

คุณไม่จำเป็นต้องสนใจว่าเครื่องมือของคุณใช้ตัวไหน เพียงแค่พิมพ์คำสั่ง Slash และมันก็ทำงานได้ แต่การทราบว่าไฟล์เหล่านี้มีอยู่ช่วยได้เมื่อเกิดปัญหา: หากคำสั่งของคุณหายไป มักจะหมายความว่าไฟล์เหล่านี้หายไปหรือล้าสมัย และ `openspec update` จะสร้างไฟล์เหล่านี้ใหม่โดยอัตโนมัติ

ดูตำแหน่งไฟล์ที่แม่นยำตามเครื่องมือได้ที่ [เครื่องมือที่รองรับ](supported-tools.md) และดูวิธีการที่สกิลแทนที่วิธีการใช้คำสั่งเพียงอย่างเดียวที่เก่ากว่าได้ที่ [คู่มือการย้ายข้อมูล](migration-guide.md)

## การยืนยันการติดตั้ง

การตรวจสอบอย่างรวดเร็ว เรียงลำดับจากเร็วที่สุดก่อน:
1. **พิมพ์เครื่องหมาย Slash ในแชทผู้ช่วย AI ของคุณ** เริ่มพิมพ์ `/opsx` และดูคำแนะนำในการเติมคำอัตโนมัติ หากคำแนะนำเหล่านี้ปรากฏขึ้น แสดงว่าคุณพร้อมใช้งานแล้ว
2. **ค้นหาไฟล์** สำหรับ Claude Code ตรวจสอบว่าโฟลเดอร์ `.claude/skills/` มีโฟลเดอร์ `openspec-*` อยู่ เครื่องมืออื่นๆ ใช้ไดเร็กทอรีของตัวเอง ([เครื่องมือที่รองรับ](supported-tools.md) มีรายชื่อไว้)
3. **รันการตั้งค่าใหม่** จากรากโปรเจกต์ของคุณ รัน `openspec update` คำสั่งนี้จะสร้างไฟล์สกิลและคำสั่งใหม่สำหรับเครื่องมือทั้งหมดที่คุณตั้งค่าไว้
4. **รีสตาร์ทผู้ช่วยของคุณ** เครื่องมือหลายตัวจะสแกนสกิลและคำสั่งเมื่อเริ่มต้นการทำงาน ดังนั้นหน้าต่างใหม่อาจเป็นขั้นตอนที่ขาดหายไป

## ฉันมีคำสั่งอะไรบ้าง?

ตามค่าเริ่มต้น OpenSpec จะติดตั้งชุดคำสั่ง Slash **หลัก** ดังนี้:
- `/opsx:explore`: คิดไอเดียกับ AI ก่อนที่จะเริ่มทำการเปลี่ยนแปลง (เป็นขั้นตอนแรกที่ดีหากคุณไม่แน่ใจ)
- `/opsx:propose`: สร้างการเปลี่ยนแปลงและร่างอาร์ติแฟกต์การวางแผนทั้งหมดในขั้นตอนเดียว
- `/opsx:apply`: สร้างการเปลี่ยนแปลงโดยทำงานตามรายการงานของมัน
- `/opsx:sync`: ผสานการอัปเดตสเปคของการเปลี่ยนแปลงเข้ากับสเปคหลักของคุณ (โดยปกติจะทำโดยอัตโนมัติ)
- `/opsx:archive`: สิ้นสุดการเปลี่ยนแปลงและเก็บไฟล์ไว้

จังหวะการทำงานที่ดีตามค่าเริ่มต้น: ใช้ `explore` เมื่อคุณกำลังหาวิธีที่จะทำอะไรต่อไป แล้วตามด้วย `propose` `apply` `archive` คู่มือ [สำรวจก่อน](explore.md) อธิบายว่าเหตุใดขั้นตอนเปิดนี้คุ้มค่า

นอกจากนี้ยังมีชุด **คำสั่งเพิ่มเติม** สำหรับผู้ที่ต้องการควบคุมได้ละเอียดยิ่งขึ้น (`/opsx:new`, `/opsx:continue`, `/opsx:ff`, `/opsx:verify`, `/opsx:bulk-archive`, `/opsx:onboard`) คุณเปิดใช้ชุดนี้ด้วย `openspec config profile` แล้วนำไปใช้ด้วย `openspec update`

ใหม่กับทั้งหมดนี้? `/opsx:onboard` (อยู่ในชุดเพิ่มเติม) จะนำคุณผ่านการเปลี่ยนแปลงทั้งหมดบนโค้ดเบสของคุณเอง โดยอธิบายแต่ละขั้นตอน เป็นการแนะนำที่เป็นมิตรที่สุดที่เป็นไปได้

สำหรับรายละเอียดการทำงานของแต่ละคำสั่ง ดูที่ [คำสั่ง](commands.md) สำหรับเวลาที่ควรใช้คำสั่งใด ดูที่ [เวิร์กโฟลว์](workflows.md)

## การรันครั้งแรกที่สะอาดและชัดเจน

รวมทั้งหมดไว้ นี่คือลำดับขั้นตอนทั้งหมดพร้อมระบุว่าแต่ละขั้นตอนเกิดขึ้นที่ใด

```text
เทอร์มินัล   $ npm install -g @fission-ai/openspec@latest
เทอร์มินัล   $ cd your-project
เทอร์มินัล   $ openspec init
              (ติดตั้งคำสั่ง Slash ลงในเครื่องมือ AI ของคุณ)

แชท AI      /opsx:explore
              (ไม่จำเป็น: คิดไอเดียกับ AI ก่อน)

แชท AI      /opsx:propose add-dark-mode
              (AI เขียนร่างข้อเสนอ สเปค การออกแบบ และรายการงาน)

แชท AI      /opsx:apply
              (AI สร้างงาน โดยตรวจสอบรายการงานทั้งหมด)

แชท AI      /opsx:archive
              (การเปลี่ยนแปลงถูกผสานเข้ากับสเปคของคุณและเก็บไฟล์ไว้)
```

สองขั้นตอนในเทอร์มินัลเพื่อตั้งค่า หลังจากนั้นคุณจะใช้งานส่วนใหญ่ในแชท นั่นคือจังหวะการทำงาน
