# คำสั่งทำงานอย่างไร

**สิ่งเดียวที่ต้องรู้คือ: OpenSpec มีคำสั่งสองประเภท และมันทำงานในสถานที่ที่แตกต่างกัน**

- คำสั่ง `openspec ...` ทำงานใน **เทอร์มินัล** ของคุณ (ตัวอย่างเช่น: `openspec init`)
- คำสั่ง `/opsx:...` ทำงานใน **แชทของผู้ช่วย AI** ของคุณ (ตัวอย่างเช่น: `/opsx:propose`)

หากคุณพิมพ์ `/opsx:propose` ลงในเทอร์มินัลของคุณแล้วไม่มีอะไรเกิดขึ้น หน้านี้คือคำตอบ สาเหตุที่คุณกำลังคุยกับ OpenSpec ผิดส่วน คำสั่งสแลชไม่ใช่คำสั่งของเทอร์มินัล พวกมันคือคำแนะนำที่คุณมอบให้กับผู้ช่วย AI สำหรับการเขียนโค้ด ในกล่องแชทเดียวกับที่คุณพิมพ์ "เพิ่มฟอร์มเข้าสู่ระบบ" ตามปกติ

ความแตกต่างเพียงเล็กน้อยนี้เป็นอุปสรรคบบ่อยที่สุดสำหรับผู้ใช้ใหม่ ดังนั้นเรามาทำให้มันชัดเจนอย่างยิ่งกันเถอะ

## สองส่วนประกอบ

OpenSpec คือโปรเจกต์หนึ่งที่สวมสองบทบาท

**CLI (ส่วนของเทอร์มินัล)** โปรแกรมชื่อ `openspec` ที่คุณติดตั้งและรันจากเชลล์ มันจะตั้งค่าโปรเจกต์ของคุณ แสดงรายการและตรวจสอบการเปลี่ยนแปลง แสดงแดชบอร์ด และเก็บถาวงานที่เสร็จสมบูรณ์ คุณพิมพ์สิ่งเหล่านี้ลงใน iTerm, VS Code terminal, PowerShell หรือที่ใดก็ตามที่คุณรัน `git` หรือ `npm`

```bash
openspec init        # ตั้งค่า OpenSpec ในโปรเจกต์นี้
openspec list        # ดูการเปลี่ยนแปลงที่ใช้งานอยู่
openspec view        # เปิดแดชบอร์ดแบบโต้ตอบได้ (interactive)
```

**คำสั่งสแลช (ส่วนของแชท)** คำสั่งสั้นๆ เช่น `/opsx:propose` และ `/opsx:apply` ที่คุณพิมพ์ลงในผู้ช่วย AI ของคุณ สิ่งเหล่านี้จะบอกให้ AI ทำตามเวิร์กโฟลว์ของ OpenSpec: ร่างข้อเสนอ เขียนสเปค สร้างจากรายการงาน เก็บถาวเมื่อเสร็จสิ้น คุณพิมพ์สิ่งเหล่านี้ลงใน Claude Code, Cursor, Windsurf, Copilot หรือผู้ช่วยที่คุณใช้

```text
/opsx:propose add-dark-mode    (พิมพ์ในแชท AI ของคุณ)
/opsx:apply                    (พิมพ์ในแชท AI ของคุณ)
/opsx:archive                  (พิมพ์ในแชท AI ของคุณ)
```

นี่คือแบบจำลองทางความคิดในภาพเดียว:

```text
        YOUR TERMINAL                         YOUR AI ASSISTANT'S CHAT
   ┌──────────────────────┐               ┌──────────────────────────────┐
   │  $ openspec init     │   installs    │  /opsx:propose add-dark-mode  │
   │  $ openspec list     │  ──────────►  │  /opsx:apply                  │
   │  $ openspec view     │   commands    │  /opsx:archive                │
   └──────────────────────┘    & skills   └──────────────────────────────┘
        run openspec here                       run /opsx:* here
```

สังเกตลูกศรุ่ง การรัน `openspec init` ในเทอร์มินัลของคุณคือสิ่งที่ *ติดตั้ง* คำสั่งสแลชลงในเครื่องมือ AI ของคุณ ส่วนของเทอร์มินัลจึงเป็นการตั้งค่าส่วนของแชท หลังจากนั้นการขับเคลื่อนงานประจำวันส่วนใหญ่จะเกิดขึ้นในแชท

## "ฉันจะเริ่มโหมดแบบโต้ตอบ (interactive mode) ได้อย่างไร?"

**ไม่มีโหมดแบบโต้ตอบแยกต่างหาก็ต้องเริ่มต้น คำถามนี้ถูกถามบ่อยมาก ดังนั้นจึงสมควรได้รับคำตอบที่ชัดเจน**

คุณไม่ได้เข้าสู่โหมด OpenSpec พิเศษณ์ คุณแค่เปิดผู้ช่วย AI สำหรับการเขียนโค้ดตามปกติ และพิมพ์คำสั่งสแลชลงในแชท คำสั่งสแลช *คือ* วิธีที่คุณ "เข้าสู่" OpenSpec ผู้ช่วยของคุณจะรับรู้ถึงมัน โหลดสกิล (skill) ของ OpenSpec ที่เกี่ยวข้อง และเริ่มทำตามเวิร์กโฟลว์

ดังนั้นคำแนะนำที่แท้จริงคือ:

1. เปิดผู้ช่วย AI สำหรับการเขียนโค้ดของคุณ (Claude Code, Cursor, Windsurf และอื่นๆ) ในโปรเจกต์
2. พิมพ์ `/opsx:propose` ลงในแชทของมัน ณ จุดเดียวกับที่คุณพิมพ์คำขออื่นใดๆ
3. สังเกตฟังก์ชันเติมข้อความอัตโนมัติ (autocomplete): หาก OpenSpec ถูกติดตั้งแล้ว คุณจะเห็น `/opsx:propose`, `/opsx:apply` และเพื่อนๆ ปรากฏขึ้นขณะที่คุณพิมพ์สแลช

แค่นั้นแหละ ไม่ต้องมีโหมดให้สลับ ไม่มีเดมอน (daemon) ให้เปิด หน้าต่างแยกต่างหาก็ไม่จำเป็น

สิ่งหนึ่งที่ *เป็นแบบโต้ตอบจริงๆ* อยู่ในเทอร์มินัลคือ `openspec view` มันจะเปิดแดชบอร์ดสำหรับการเรียกดูสเปคและการเปลี่ยนแปลงของคุณ แต่นั่นเป็นเครื่องมือแสดงผล ไม่ใช่สิ่งที่ใช้ในการเสนอและสร้างสรรค์ การสร้างเกิดขึ้นผ่านคำสั่งสแลชในแชท

## เหตุใดจึงมีการแบ่งแยกเช่นนี้

มันคุ้มค่าที่จะทำความเข้าใจ เพราะมันอธิบายว่าทำไม่อะไรถึงทำงานร่วมกับเครื่องมือ AI ที่แตกต่างกันกว่า 25 ตัว

CLI คือ **กลไก (engine)** มันรู้กฎ: โฟลเดอร์การเปลี่ยนแปลงมีหน้าตาอย่างไร อาร์ติแฟคต์ใดขึ้นอยู่กับอะไร วิธีที่จะรวม delta spec เข้ากับแหล่งความจริง (source of truth) ของคุณ มันเหมือนกันในทุกที่

คำสั่งสแลชคือ **พวงมาลัย** และเครื่องมือ AI ทุกตัวก็มีรูปแบบของมันเล็กน้อย Claude Code เรียกสิ่งเหล่านี้ว่าคำสั่ง Cursor และ Windsurf มีรูปแบบของตัวเอง บางเครื่องมือเรียกสิ่งเหล่านี้ว่าสกิล (skills) เมื่อคุณรัน `openspec init` OpenSpec จะสร้างไฟล์ที่ถูกต้องสำหรับแต่ละเครื่องมือที่คุณเลือก ดังนั้นเจตนา `/opsx:propose` เดียวกันจึงทำงานได้ไม่ว่าคุณจะชอบผู้ช่วยคนไหน

จุดแข็งของการออกแบบนี้คือ: คุณเรียนรู้เวิร์กโฟลว์เพียงครั้งเดียวและนำไปใช้กับเครื่องมือต่างๆ ข้อแลกเปลี่ยนคือ: ไวยากรณ์ที่แน่นอนของคำสั่งอาจแตกต่างกันเล็กน้อยระหว่างเครื่องมือ ซึ่งเป็นส่วนถัดไป

## รูปแบบไวยากรณ์ของคำสั่งสแลชตามเครื่องมือ

เจตนา (intent) นั้นเหมือนกันในทุกที่ แต่เครื่องหมายวรรคตอนนั้นต่างกัน ใช้รูปแบบที่ตรงกับผู้ช่วยของคุณ

| Tool | How you type it |
|------|-----------------|
| Claude Code | `/opsx:propose`, `/opsx:apply` |
| Cursor | `/opsx-propose`, `/opsx-apply` |
| Windsurf | `/opsx-propose`, `/opsx-apply` |
| GitHub Copilot (IDE) | `/opsx-propose`, `/opsx-apply` |
| Kimi CLI | skill-style, e.g. `/skill:openspec-propose` |
| Trae | skill-style, e.g. `/openspec-propose` |

เครื่องมือส่วนใหญ่ใช้รูปแบบโคลอน (`/opsx:propose`) หรือรูปแบบขีดกลาง (`/opsx-propose`) เครื่องมือบางตัวแสดง OpenSpec เป็นสกิลที่มีชื่อแทนที่จะเป็นคำสั่งสแลช สำหรับสิ่งเหล่านั้นคุณต้องเรียกใช้งานสกิลด้วยชื่อเต็ม รายการทั้งหมดต่อเครื่องมือ รวมถึงไฟล์ใดบ้างที่ถูกเขียนไว้ที่ไหน อยู่ใน [Supported Tools](supported-tools.md)

เมื่อไม่แน่ใจ ให้พิมพ์สแลชในแชท AI ของคุณและดูฟังก์ชันเติมข้อความอัตโนมัติ เครื่องมือของคุณจะแสดงรูปแบบที่มันคาดหวัง

## คำสั่งมาได้อย่างไร: สกิลและคำสั่ง

เมื่อคุณรัน `openspec init` (หรือ `openspec update`) OpenSpec จะเขียนไฟล์ขนาดเล็กๆ ลงในโปรเจกต์ของคุณเพื่อให้เครื่องมือ AI ของคุณสามารถค้นหาเวิร์กโฟลว์ได้ ขึ้นอยู่กับเครื่องมือและการตั้งค่าของคุณ สิ่งเหล่านี้คือ **สกิล (skills)** **คำสั่ง (commands)** หรือทั้งสองอย่าง

- **Skills** อยู่ในที่เช่น `.claude/skills/openspec-*/SKILL.md` พวกมันเป็นมาตรฐานข้ามเครื่องมือที่กำลังเกิดขึ้น: โฟลเดอร์ของคำแนะนำที่ผู้ช่วยของคุณตรวจจับโดยอัตโนมัติ
- **Commands** อยู่ในที่เช่น `.claude/commands/opsx/<id>.md` พวกมันคือไฟล์คำสั่งสแลชแบบเฉพาะเจาะจงสำหรับแต่ละเครื่องมือที่เป็นรุ่นเก่า

คุณไม่จำเป็นต้องสนใจว่าเครื่องมือของคุณใช้สิ่งใด การพิมพ์คำสั่งสแลชแล้วใช้งานได้ก็พอ แต่การรู้ว่าไฟล์เหล่านี้มีอยู่ช่วยได้เมื่อมีบางอย่างผิดพลาด: หากคำสั่งของคุณหายไป โดยปกติหมายความว่าไฟล์เหล่านี้ขาดหายหรือล้าสมัย และ `openspec update` จะสร้างมันขึ้นใหม่

ดู [Supported Tools](supported-tools.md) สำหรับเส้นทางที่แน่นอนต่อเครื่องมือ และ [Migration Guide](migration-guide.md) สำหรับวิธีการที่สกิลเข้ามาแทนที่แนวทางแบบคำสั่งเท่านั้นที่เป็นรุ่นเก่า

## การยืนยันว่าติดตั้งแล้ว

การตรวจสอบอย่างรวดเร็ว เร็วที่สุดก่อน:

1. **พิมพ์สแลชในแชท AI ของคุณ** เริ่มพิมพ์ `/opsx` และมองหาข้อเสนอแนะจากฟังก์ชันเติมข้อความอัตโนมัติ หากปรากฏขึ้น แสดงว่าพร้อม
2. **มองหาสิ่งที่เกี่ยวข้องกับไฟล์** สำหรับ Claude Code ให้ตรวจสอบว่า `.claude/skills/` มีโฟลเดอร์ `openspec-*` เครื่องมืออื่นๆ ใช้ไดเรกทอรีของตัวเอง ([Supported Tools](supported-tools.md) จะแสดงรายการให้)
3. **รันการตั้งค่าซ้ำ** จากรูทโปรเจกต์ ให้รัน `openspec update` สิ่งนี้จะสร้างไฟล์สกิลและคำสั่งขึ้นใหม่สำหรับเครื่องมือที่คุณกำหนดค่าไว้
4. **รีสตาร์ทผู้ช่วยของคุณ** เครื่องมือจำนวนมากสแกนหาสกิลและคำสั่งเมื่อเริ่มต้น ดังนั้นหน้าต่างที่สดใหมอาจเป็นขั้นตอนที่ขาดหายไป

## ฉันมีคำสั่งอะไรบ้าง?

โดยค่าเริ่มต้น OpenSpec จะติดตั้งชุดคำสั่งสแลช **หลัก (core)**:

- `/opsx:explore`: คิดวิเคราะห์แนวคิดกับ AI ก่อนที่จะตัดสินใจเปลี่ยนแปลงใดๆ (เป็นขั้นตอนแรกที่ดีมากเมื่อคุณไม่แน่ใจ)
- `/opsx:propose`: สร้างการเปลี่ยนแปลงและร่างอาร์ติแฟกต์การวางแผนทั้งหมดในขั้นตอนเดียว
- `/opsx:apply`: สร้างการเปลี่ยนแปลงโดยทำงานผ่านรายการงานของมัน
- `/opsx:sync`: รวมการอัปเดตสเปคของการเปลี่ยนแปลงเข้ากับสเปคหลักของคุณ (โดยปกติเป็นอัตโนมัติ)
- `/opsx:archive`: สิ้นสุดการเปลี่ยนแปลงและจัดเก็บไฟล์

จังหวะเริ่มต้นที่ดีคือ: `explore` เมื่อคุณกำลังพิจารณาว่าจะทำอะไร จากนั้น `propose`, `apply`, `archive` คู่มือ [Explore First](explore.md) อธิบายว่าเหตุใดขั้นตอนเปิดนี้จึงคุ้มค่า

ยังมีชุด **ขยาย (expanded)** สำหรับผู้ที่ต้องการการควบคุมที่ละเอียดกว่า (`/opsx:new`, `/opsx:continue`, `/opsx:ff`, `/opsx:verify`, `/opsx:bulk-archive`, `/opsx:onboard`) คุณเปิดใช้งานด้วย `openspec config profile` จากนั้นใช้ `openspec update`

เพิ่งเริ่มสนใจเรื่องนี้? `/opsx:onboard` (ในชุดขยาย) จะนำคุณไปตลอดการเปลี่ยนแปลงที่สมบูรณ์บนโค้ดเบสของคุณ โดยบรรยายแต่ละขั้นตอน มันคือบทแนะนำที่เป็นมิตรที่สุดเท่าที่จะเป็นไปได้

สำหรับรายละเอียดว่าแต่ละคำสั่งทำอะไร ดู [Commands](commands.md) สำหรับเมื่อไหร่ควรใช้สิ่งใด ดู [Workflows](workflows.md)

## การรันครั้งแรกอย่างสะอาดตา

สรุปทั้งหมด นี่คือลำดับทั้งหมดโดยระบุขั้นตอนแต่ละขั้นตอนว่าเกิดขึ้นที่ไหน

```text
TERMINAL   $ npm install -g @fission-ai/openspec@latest
TERMINAL   $ cd your-project
TERMINAL   $ openspec init
              (ติดตั้งคำสั่งสแลชลงในเครื่องมือ AI ของคุณ)

AI CHAT      /opsx:explore
              (ไม่บังคับ: คิดวิเคราะห์แนวคิดกับ AI ก่อน)

AI CHAT      /opsx:propose add-dark-mode
              (AI ร่างข้อเสนอ สเปค การออกแบบ งานต่างๆ)

AI CHAT      /opsx:apply
              (AI สร้างมัน โดยการทำเครื่องหมายรายการงานให้เสร็จสิ้น)

AI CHAT      /opsx:archive
              (การเปลี่ยนแปลงถูกรวมเข้ากับสเปคของคุณและจัดเก็บไว้)
```
