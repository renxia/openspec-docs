# Багатомовний посібник

Налаштуйте OpenSpec для генерування артефактів мовами, відмінними від англійської.

## Швидке налаштування

Додайте інструкцію з мови до файлу `openspec/config.yaml`:

```yaml
schema: spec-driven

context: |
  Мова: Португальська (pt-BR)
  Усі артефакти мають бути написані бразильською португальською мовою.

  # Ваш інший контекст проекту нижче...
  Технологічний стек: TypeScript, React, Node.js
```

Це все. Тепер усі згенеровані артефакти будуть португальською мовою.

## Приклади мов

### Португальська (Бразилія)

```yaml
context: |
  Мова: Португальська (pt-BR)
  Усі артефакти мають бути написані бразильською португальською мовою.
```

### Іспанська

```yaml
context: |
  Idioma: Español
  Todos los artefactos deben escribirse en español.
```

### Китайська (спрощена)

```yaml
context: |
  语言：中文（简体）
  所有产出物必须用简体中文撰写。
```

### Японська

```yaml
context: |
  言語：日本語
  すべての成果物は日本語で作成してください。
```

### Французька

```yaml
context: |
  Langue : Français
  Tous les artefacts doivent être rédigés en français.
```

### Німецька

```yaml
context: |
  Sprache: Deutsch
  Alle Artefakte müssen auf Deutsch verfasst werden.
```

## Поради

### Обробка технічних термінів

Вирішіть, як обробляти технічну термінологію:

```yaml
context: |
  Мова: Японська
  Писати японською, але:
  - Технічні терміни, такі як "API", "REST", "GraphQL", залишати англійською
  - Приклади коду та шляхи до файлів залишати англійською
```

### Поєднання з іншим контекстом

Налаштування мови працюють разом з іншим контекстом вашого проекту:

```yaml
schema: spec-driven

context: |
  Мова: Португальська (pt-BR)
  Усі артефакти мають бути написані бразильською португальською мовою.

  Технологічний стек: TypeScript, React 18, Node.js 20
  База даних: PostgreSQL з Prisma ORM
```

## Перевірка

Щоб перевірити, чи працює ваша мовна конфігурація:

```bash
# Перевірте інструкції - має показати ваш мовний контекст
openspec instructions proposal --change my-change

# Вивід включатиме ваш мовний контекст
```
